=== STEP 1: APP INFORMATION ===

Title: Beep.Buzz

Description: beep.buzz is a playful social media platform where users create personalized pages at their own subdomain (username.beep.buzz). Each page is a customizable, vertically-scrolling canvas containing text, shapes, and embedded YouTube videos. Users interact by placing decorative "beep" and "buzz" stickers on each other's pages, with each sticker serving as a clickable link to that visitor's page. The platform emphasizes creative expression, simple interactions, and organic discovery through sticker-based navigation.

=== STEP 2: PAGES ===

Page 1:
Name: Homepage
Route: /
Description: The main discovery and navigation hub. Users can search for pages by title or username, browse featured sections (Popular Pages, Recently Active, New Pages), filter by categories, or discover random pages. Each page card displays title, username, preview, sticker counts, and category.

Page 2:
Name: User Page
Route: /[username]
Description: Each user's personal subdomain page. Shows static header with page title, username, creator's stickers, and counts. Sticky header with sticker visibility toggle and "Add Beep/Buzz" buttons. Main content area with creator's elements and visitor stickers layered on top. Clicking stickers reveals who placed them and navigates to their page.

Page 3:
Name: Page Editor
Route: /edit
Description: Builder interface for creating and customizing personal pages. Users add text blocks, shapes, dividers, and YouTube embeds. Elements can be dragged, resized, aligned, edited, and deleted. Includes responsive preview toggles for desktop, tablet, and mobile. Enforces limits: 20 elements max, 3 YouTube embeds, 3000px height, 50 stickers per type.

Page 4:
Name: User Settings
Route: /settings
Description: Manage personal sticker identity and account. Change beep and buzz icons from available libraries, customize colors and background shapes. Changes update retroactively everywhere. Activity overview shows all pages with placed stickers, with navigation and removal options. Analytics show total stickers received with breakdowns.

Page 5:
Name: Onboarding
Route: /welcome
Description: First-time user profile setup after email magic link authentication. Choose unique username (3-20 URL-friendly characters, AI-scanned). Select and customize personal beep and buzz icons, choosing icons, colors, and shapes. Immediate access to explore and place stickers after setup.

Page 6:
Name: Admin Dashboard
Route: /admin
Description: Admin-only moderation interface. Displays queue of reported pages, list of auto-hidden pages (3+ reports), and full content viewing. Actions include approve (unhide/dismiss reports), remove permanently, warn users, and ban users. Audit log tracks all moderation actions.

Database tables:

Table 1:
Name: users
Description: Stores user account information including email, username (subdomain), join date, account status (active/banned), and role (user/admin). Contains sticker customization settings for both beep and buzz styles including selected icons, colors, and background shapes. Used for authentication, profile management, and applying user's signature sticker styles across the platform.

Table 2:
Name: pages
Description: Contains user-created page content including page title, category, subdomain/username reference, creation date, last modified date, visibility status, and view count. Stores the overall page configuration and metadata needed for discovery, search, filtering, and displaying page cards in the homepage feed.

Table 3:
Name: page_elements
Description: Stores individual content elements placed on pages including element type (text/shape/divider/youtube_embed), content data, position coordinates (x, y), dimensions (width, height), z-index for layering, styling properties, and creation order. Links to parent page and enforces limits (20 elements max, 3 YouTube embeds per page).

Table 4:
Name: stickers
Description: Records all beep and buzz stickers placed on pages including sticker type (beep/buzz), placer user reference, target page reference, coordinates on page, placement timestamp, and visibility status. Each sticker serves as both decoration and navigation link. Used for interaction tracking, sticker count analytics, and enabling click-to-navigate functionality.

Table 5:
Name: reports
Description: Tracks user-submitted content reports including reporter user reference, reported page reference, report reason/category, additional notes, submission timestamp, and resolution status (pending/dismissed/actioned). Enables moderation queue, auto-hiding pages at 3+ reports threshold, and maintaining platform safety through community reporting.

Table 6:
Name: moderation_logs
Description: Records all administrative moderation actions including moderator user reference, affected page or user reference, action type (approve/remove/warn/ban), timestamp, and moderator notes. Maintains complete audit trail of content moderation decisions, user warnings, bans, and page removals for accountability and review purposes.
